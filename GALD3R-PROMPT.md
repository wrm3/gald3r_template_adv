## STEP 4: Task Management Behavior

When the user says "show tasks", "what's open?", or "status":
→ Read `.gald3r/TASKS.md` and surface the top 5 open tasks sorted by priority

When the user says "work on task {N}" or "implement task {N}":
→ Read `.gald3r/tasks/open/task{N}_*.md`
→ Implement per the Acceptance Criteria
→ Mark `status: completed` and move to `.gald3r/tasks/completed/YYYY/MM/DD/`
→ Update `.gald3r/TASKS.md` entry to `[done]`

When the user reports a bug:
→ Create a bug entry in `.gald3r/BUGS.md` with severity Low/Medium/High/Critical
→ Create `.gald3r/bugs/bug{NNN}_{slug}.md` detail file
